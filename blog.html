Frequently Asked Questions

Local Storage vs Session Storage
▪ Local Storage can store around 10 mb information, where the limit of Session Storage is naturally 5 mb.
▪ Local Storage can survive refresh, and restart. It is accessible from any window. ButSession Storage is accessible only from that tab of browser. It also can survive refresh, and restart.
▪ Local Storage expires only when 'localStorage.clear()' method is called. Session Storage also expires when 'localStorage.clear()' is called. Besides, it expires when the current tab/session of browser is closed.
Global Scope vs Block Scope
The main difference between Global Scope and Block Scope is :

▪ Global Scope id Accessed globally, when a variable is declared globally, is not in any function or block, it has a global scope. It can be accessed by any JavaScript program and function.
▪ On the other hand, Block Scope can be accessed by only in self and child blocks. A function contains a block, any variable declared inside a function has Function Scope which acts just like the Block Scope. In a function or a block o codes, Block Scopes can't be accessed by outside the block, but can be accessed in any child block.
Javascript Event Loop
▪ The event loop is the secret behind JavaScript's asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.
▪ The call stack is responsible for keeping track of all the operations in line to be executed. Whenever a function is finished, it is popped from the stack.
▪ The event queue is responsible for sending new functions to the stack for processing. It follows the queue data structure to maintain the correct sequence in which all operations should be sent for execution.
▪ Whenever an async function is called, it is sent to a browser API. These are APIs built into the browser. Based on the command received from the call stack, the API starts its own single-threaded operation.
How can we get 'undefined' in JavaScript?
▪ Setting a value undefined willingly.
▪ Variables which are declared but not have been set a value, returns undefined.
▪ Function without a return or with a return command but with no return value will return undefined by default.
▪ Parameters that are not passed through a function will define that argument as undefined.
▪ While trying to access an array element, if index is out of range, undefined is returned.
▪ Deleting an array element from that array will create an undefined value inside that array with that element index.
▪ When object value / property is not found, it will return undefined.
